## Decisions on Ambiguities

Before the manifest, these decisions resolve the flagged ambiguities so each bead is fully specified:

**AMB-1 (Mixed file list in session.open):** Adopt the recommendation from the understanding report:
reject external/advisory files in `session.open` with a clear error that includes the handler guidance
(`"path report.xlsx is managed by the sheets handler — use sheets_session(\"open ...\") instead"`).
Text-only calls continue unchanged. This avoids redesigning `SessionOpenResult`.

**AMB-2 (Digest on mutation responses):** Use Option B: serialize result to `serde_json::Value` and
insert `"session_digest"` as a top-level key. The `dispatch()` function gets a post-processing step
that wraps mutation responses. No changes to the existing result structs. The `file.read` and
`cursor.move` responses are NOT wrapped.

**AMB-5 (Sweeper/Tracker sync):** The sweeper in `main.rs` currently ignores swept session IDs.
Update the sweeper loop to call `coordinator.on_sessions_swept(&expired)` after each sweep. The
`Coordinator` method marks all `Native` and `Advisory` entries whose `session_id` matches as `Closed`.

**AMB-3 (coordinator.check action vocabulary):** Only `"build"` is supported in this phase.
Unknown action names return an error (`"unknown action: {name}"`), not a no-op.

**AMB-4 (Tracking ID generation):** Use a `AtomicU64` counter in `Coordinator`. IDs are formatted
as `"ext-{n:03}"` for the first 999, plain `"ext-{n}"` beyond that. Thread-safe increment on each
registration.

**AMB-6 (file.write warning for externally-tracked path):** The warning is embedded in the
`session_digest` field that is appended to the write response. The `DigestEntry` for the
externally-managed file will carry an `advisory` field: `"file.write to externally-managed path"`.
The write still succeeds; the advisory is non-blocking.

### 2.1 — `registry.rs`: FormatRegistry Implementation

**File to create:** `crates/slipstream-daemon/src/registry.rs`

Full implementation of:

1. `FormatRegistry::from_config(config: RegistryConfig) -> Self`
   - Iterates all handlers, builds `ext_map: HashMap<String, String>` from extension → handler name.
   - Extensions are lowercased during indexing.
   - Also indexes exact filenames (Makefile, Dockerfile) into the same `ext_map`.

2. `FormatRegistry::default_registry() -> Self`
   - Constructs `RegistryConfig` from hardcoded Rust literals (matching the TOML in the design doc).
   - Handlers: sheets (xlsx, xls), drawio (drawio, dio), midi (mid, midi), terraform (tf, tfvars),
     make (Makefile, makefile, GNUmakefile), docker (Dockerfile).
   - Calls `from_config`.

3. `FormatRegistry::load(path: &Path) -> Self`
   - Reads file at `path` with `std::fs::read_to_string`.
   - Parses with `toml::from_str::<RegistryConfig>`.
   - On any error (IO or parse), logs a warning and returns `default_registry()`.

4. `FormatRegistry::lookup_ext(&self, ext: &str) -> Option<&HandlerEntry>`
   - Lowercases ext, looks up in `ext_map`, returns corresponding `HandlerEntry` ref.

5. `FormatRegistry::lookup_filename(&self, filename: &str) -> Option<&HandlerEntry>`
   - Tries `ext_map.get(filename)` (exact match for Makefile, Dockerfile).
   - Falls back to extracting file extension and calling `lookup_ext`.

6. `FullHandlerConfig::interpolate_path(&self, path: &Path) -> String`
   - Replaces `{path}` in `self.session_open` with `path.display().to_string()`.

### 2.2 — `coordinator.rs`: Coordinator Implementation

**File to create:** `crates/slipstream-daemon/src/coordinator.rs`

Full implementation of all methods on `Coordinator`:

1. `Coordinator::new()`
   - Creates empty `DashMap`, `AtomicU64::new(0)`.

2. Tracking ID generation helper:
   ```rust
   fn next_tracking_id(&self) -> String {
       let n = self.next_id.fetch_add(1, Ordering::Relaxed) + 1;
       if n <= 999 { format!("ext-{n:03}") } else { format!("ext-{n}") }
   }
   ```

3. `register_native(path, canonical, session_id) -> String`
   - Canonicalizes path if not already canonical.
   - Generates tracking_id via `next_tracking_id()`.
   - Inserts `TrackedFile { handler: HandlerType::Native { session_id }, state: FileState::Clean, ... }`.
   - Returns tracking_id.

4. `register_external(path, handler_name) -> Result<String, CoordinatorError>`
   - Canonicalizes path (using `std::fs::canonicalize`, fallback to `path.to_path_buf()` if file doesn't exist yet).
   - Generates tracking_id.
   - Inserts with `state: FileState::ExternallyManaged`.
   - Returns tracking_id.

5. `unregister(tracking_id) -> Result<(), CoordinatorError>`
   - Scans `files` for entry whose `tracking_id == tracking_id`.
   - Removes it. Returns `TrackingIdNotFound` if not found.

6. `mark_dirty(canonical, edit_count)`
   - If entry exists, updates `state = FileState::Dirty { edit_count }` and `last_activity`.
   - No-op if not found (graceful).

7. `mark_flushed(canonical)`
   - Updates matching entry's `state = FileState::Flushed`.

8. `mark_closed_by_session(session_id)`
   - Iterates all entries. Sets `state = FileState::Closed` for any entry whose `HandlerType` contains matching `session_id`.

9. `on_sessions_swept(expired: &[SessionId])`
   - For each session_id in `expired`, calls `mark_closed_by_session`.

10. `build_digest(cwd: &Path) -> SessionDigest`
    - Iterates all entries, counts native/dirty/external.
    - For each, builds `DigestEntry`:
      - `path`: `pathdiff::relative_path(entry.canonical_path, cwd)` if possible, else absolute.
      - `handler`: handler name string.
      - `state`: human-readable string based on `FileState`.
      - `advisory`: `Some("file.write to externally-managed path")` only if handler is External and a write advisory is pending (this is passed in via a flag or checked contextually — see handler integration).
    - Note: advisory on write conflict is injected by handler, not coordinator. The coordinator's `build_digest` does NOT auto-inject write advisories. The handler calls `build_digest` and then patches the relevant `DigestEntry` if a write to an external path was just attempted.

11. `status() -> CoordinatorStatus`
    - Populates `tracked_files` from all entries.
    - `native_sessions`: de-duplicates session IDs from Native/Advisory entries, counting dirty files per session.
    - `external_registrations`: all External entries.
    - `warnings`: native entries with `Dirty { edit_count > 0 }` → `"path has N unflushed edits"`.

12. `check_action(action: &str) -> Result<CheckResult, CoordinatorError>`
    - If `action != "build"` → `Err(UnknownAction(action.to_string()))`.
    - Collects dirty native sessions: `"path has N unflushed edits — flush session SID first"`.
    - Collects external entries: `"path is externally managed by HANDLER — ensure it's saved"`.
    - If no warnings → `suggestion = "Ready to build"`.
    - If warnings → builds suggestion string listing flush commands then save commands.
    - Returns `CheckResult { warnings, suggestion }`.

**Implementation note on `pathdiff`:** Either add `pathdiff = "0.2"` crate, or implement inline:
```rust
fn relative_or_absolute(path: &Path, base: &Path) -> String {
    pathdiff::diff_paths(path, base)
        .unwrap_or_else(|| path.to_path_buf())
        .display()
        .to_string()
}
```
Decision: add `pathdiff = "0.2"` to `slipstream-daemon/Cargo.toml`.

### 2.3 — `handler.rs`: Format-Aware Open + Coordinator Integration

**File to modify:** `crates/slipstream-daemon/src/handler.rs`

Changes required:

1. **`dispatch` signature change:**
   ```rust
   pub fn dispatch(
       req: Request,
       mgr: &Arc<SessionManager>,
       registry: &Arc<FormatRegistry>,
       coordinator: &Arc<Coordinator>,
   ) -> Response
   ```
   Add four new match arms:
   - `"coordinator.status"` → `handle_coordinator_status`
   - `"coordinator.register"` → `handle_coordinator_register`
   - `"coordinator.unregister"` → `handle_coordinator_unregister`
   - `"coordinator.check"` → `handle_coordinator_check`

2. **`handle_session_open` rewrite:**
   For each file in `params.files`:
   - Look up by filename/extension in `registry`.
   - If `HandlerEntry::Full(cfg)`: do NOT open a text buffer. Build `ExternalHandlerResult`,
     call `coordinator.register_external(&path, &handler_name)`, include tracking_id in result.
     Return the result directly (no session created for external files). If ALL files are external,
     return a `Vec<ExternalHandlerResult>` (one per file). If ANY file is external, reject the call
     with error: `"mixed session.open: path {x} is managed by {handler} — open external files separately"`.
   - If `HandlerEntry::Advisory(cfg)`: open as text buffer (existing path), also include guidance
     fields in the response. Register in coordinator as Native with advisory handler name.
   - If no match: existing behavior.

3. **Digest injection helper:**
   ```rust
   fn inject_digest(
       response: Response,
       coordinator: &Coordinator,
       cwd: &Path,
   ) -> Response
   ```
   - If `response.result` is `Some(value)`, inserts `value["session_digest"] = coordinator.build_digest(cwd)`.
   - If `response.error` is set, passes through unchanged.
   - `cwd` = `std::env::current_dir().unwrap_or_default()`.

4. **Mutation handlers that get digest injected:**
   - `handle_session_open` (native/advisory paths only)
   - `handle_session_flush` (on success, also calls `coordinator.mark_flushed`)
   - `handle_session_close` (also calls `coordinator.mark_closed_by_session`)
   - `handle_file_write` (also calls `coordinator.mark_dirty`; if path is externally tracked, adds advisory to digest)
   - `handle_file_str_replace` (also calls `coordinator.mark_dirty`)

5. **`handle_coordinator_status(req, coordinator) -> Response`**
   - No params needed (empty or `{}` accepted).
   - Calls `coordinator.status()`, serializes, wraps with `inject_digest`.

6. **`handle_coordinator_register(req, coordinator, registry) -> Response`**
   - Parses `CoordinatorRegisterParams`.
   - Verifies `handler` is known in registry (optional validation — warn if unknown but still register).
   - Calls `coordinator.register_external(&path, &handler)`.
   - Returns `CoordinatorRegisterResult { tracking_id, status: "registered" }`.
   - Injects digest.

7. **`handle_coordinator_unregister(req, coordinator) -> Response`**
   - Parses `CoordinatorUnregisterParams`.
   - Calls `coordinator.unregister(&tracking_id)`.
   - On `CoordinatorError::TrackingIdNotFound` → RPC error 404.
   - On success → `Response::ok(id, serde_json::json!({}))`.
   - Injects digest.

8. **`handle_coordinator_check(req, coordinator) -> Response`**
   - Parses `CoordinatorCheckParams`.
   - Calls `coordinator.check_action(&action)`.
   - On `CoordinatorError::UnknownAction` → RPC error `ERR_INVALID_PARAMS`.
   - On success → `Response::ok(id, check_result)` with digest.

9. **Sweeper coordination:** The sweeper loop in `main.rs` must pass swept IDs to coordinator.
   The `serve()` function signature must be extended or the sweeper must be wired post-serve.
   (See Integration section 4.1.)

## Gaps and Underspecified Items

The following items were NOT explicitly specified in the phase doc and decisions have been made:

1. **`pathdiff` crate for relative paths in digest**: The doc says "relative to CWD if possible" but doesn't specify how. Decision: add `pathdiff = "0.2"` crate.

2. **`dirs` crate vs inline for config path**: The doc says `~/.config/slipstream/formats.toml` but doesn't specify how to resolve `~`. Decision: inline without extra crate, use `$HOME` env var.

3. **`Coordinator` as `Arc<Coordinator>` vs `Arc<Mutex<Coordinator>>`**: `DashMap` provides interior mutability; `AtomicU64` provides interior mutability. `Coordinator` can use `&self` for all methods. No `Mutex` needed.

4. **`on_sessions_swept` takes `&[SessionId]`**: The sweeper currently returns `Vec<SessionId>`. The method signature mirrors this exactly.

5. **`register_native` called at open time, not lazily**: The handler calls `coordinator.register_native` immediately when the text buffer is successfully created, before returning the `SessionOpenResult`.

6. **Advisory files registered with `HandlerType::Advisory`**: Tracked in coordinator with `session_id` so they participate in sweeper cleanup. State starts as `Clean`.

7. **`handle_session_open` with all-external file list**: Returns a `Vec<ExternalHandlerResult>` (array), not a single object. This is consistent with the plural `files` param.

8. **Digest injection on `handle_batch`**: The `batch` handler applies multiple mutations. After the batch completes, the digest is built once and injected. (Not per-op.)

9. **`slipstream_exec` and coordinator**: `slipstream_exec` calls `session.open`, batch, flush, close internally. Each of those will trigger tracker updates as normal. The final response from `exec` is the aggregated output; the `close` sub-call will return a digest, but `exec` wraps the responses in an outer object. Decision: the `exec` MCP tool wraps sub-results as-is; the `close` step's `session_digest` will be present inside `output["close"]["session_digest"]`. The top-level `exec` result does not add another digest. This avoids duplicating digest injection logic. Document in tool description.

10. **`file.str_replace` in `batch` ops updates coordinator**: The `handle_batch` calls `session.str_replace` which doesn't separately update the coordinator. The handler must call `coordinator.mark_dirty` after the batch completes if any `StrReplace` or `Write` ops were in the batch. Since batch operates on a single session, one call to `mark_dirty` with the final `edits_pending` count is sufficient.
