- **ss-0zo.1 and ss-0zo.4** both target `registry.rs` — .1 adds type definitions, .4 adds method bodies. An agent can do them together or sequentially.
- **ss-0zo.2 and ss-0zo.5** similarly target `coordinator.rs` as type definitions then implementations.

- **ss-0zo.7** (wiring) must come after ss-0zo.6 since it updates the serve() signature to match the new dispatch() signature.
- Test beads (ss-0zo.8 through ss-0zo.12) should be done after their respective implementation beads are complete.
- **ss-0zo.13** (MCP tools) can be done in parallel with daemon work since it builds on top of the RPC layer.
